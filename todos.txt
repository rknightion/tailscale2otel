0. WHAT THIS PROJECT IS
--------------------------------------------------------------------------------
A lightweight Go service that polls the central Tailscale API
(api.tailscale.com/api/v2) for every available kind of observability data and
exports it as OTEL-native metrics + logs over OTLP. Optimized for Grafana Cloud
but works with any OTEL backend. Reference inspiration: ktranslate
(/Users/rob/repos/ktranslate) — sflow→OTEL; we apply the same pipeline shape to
the Tailscale API.

STATUS: Core implementation complete and green. `go build ./...`,
`go vet ./...`, and `go test -race ./...` all pass across 21 packages
(68 .go files, 31 test files, ~132 test funcs). Runtime smoke verified: the
binary starts and exports `tailscale2otel.up` via OTLP/stdout with correct
Resource + scope. NOT yet committed (branch: feat/initial-implementation).
NOT yet run against a real tailnet or Grafana Cloud.

2. DESIGN DECISIONS ALREADY MADE (do not relitigate without reason)
--------------------------------------------------------------------------------
User decisions:
  - Language: Go (single static binary).
  - Flow logs modeled as BOTH aggregated metrics AND full-fidelity OTEL logs.
  - Export: OTLP push (gRPC/HTTP); stdout protocol for local debug.
  - Scope: ALL Tailscale data sources.
  - Webhook receiver: built, OFF by default.
  - Flow-log default log verbosity: per_connection (configurable: per_record | off).
  - Module/org: github.com/rknightion/tailscale2otel.
  - Both ingestion methods for flow+audit logs: POLL and STREAM (Splunk-HEC-style
    receiver). S3/GCS object-reader ingestion was DEFERRED (not built).
  - Target Go 1.26 and use its features where helpful.

Engineering decisions:
  - D1 Hybrid Tailscale client: official tsclient for snapshot resources; a thin
    custom HTTP doer (internal/tsapi/logs.go) for the two /logging endpoints the
    client does not expose. Both share one authenticated, retrying *http.Client.
  - D2 Use the official OTEL log SDK (NOT ktranslate's third-party logs lib).
  - D3 Flow-log "tail" hazard: never poll up to now; always poll [checkpoint,
    now-lag]. nextWindow() in internal/collector/window.go implements cold-start
    lookback, lag, and max-window capping.
  - D4 Device identity = METRIC ATTRIBUTES (host.name/host.id/os.type/...), not a
    per-device Resource. (Per-device Resource is a possible v2 change.)
  - D5 Synchronous instruments (Float64Counter.Add / Float64Gauge.Record), not
    ktranslate's async observable-gauge+channel design.
  - D7 Checkpoint store: in-memory (default) + file-backed (opt-in via
    checkpoint.store: file). Atomic file writes (temp+rename).
  - D8 Streaming ~= polling in content; the receiver emulates Splunk HEC
    (POST /services/collector/event, `Authorization: Splunk <token>`). Tailscale
    does NOT document the exact HEC payload envelope -> the parser is defensive
    and the envelope MUST be pinned empirically by capturing a live stream.
  - OTEL log SDK v0.6.0 has NO Record.EventName field; the event name is carried
    as the "event.name" attribute on each log record. SWITCH to the native field
    when the SDK adds it.
  - semconv-compliant naming: dotted lowercase, no `_total` suffix, UCUM units
    (By, {packet}, {flow}, {event}, s, d, 1). Low-cardinality dims on metrics;
    full 5-tuple / high-cardinality only on log records.

3. HARD CONSTRAINTS DISCOVERED IN THE PINNED tailscale-client-go/v2
   (these forced deviations; verify with `go doc` before assuming otherwise)
--------------------------------------------------------------------------------
- tsclient.Device is FLAT and has NO ClientConnectivity field => NO DERP-latency
  data source in this version. `tailscale.device.derp.latency` is NOT emitted.
  (If the dep is upgraded to a Device with ClientConnectivity, add it back.)
- Device has NO "connectedToControl" flag => `tailscale.device.online` is derived
  from LastSeen recency within a window (default 5m) using an injectable clock
  (devices/export_test.go SetClock seam).
- Device field is `ID` (not `NodeID`); mapped to DeviceMeta.NodeID + host.id.
- Device has NO OSVersion; left "".
- Device timestamps (LastSeen/Created/Expires) are value type `tsclient.Time`
  (embeds time.Time); guard with .IsZero(), read .Unix().
- tsclient.Key has NO KeyType field => key "type" is derived from Capabilities
  (ephemeral|reusable|onetime). Key HAS Revoked (time.Time) and Invalid (bool).
- User.LastSeen and Key.Expires are plain time.Time (not the wrapper, not pointers).
- User.Role/Status/Type are defined string types (convert via string()).
- TailnetSettings bool fields that EXIST: DevicesApprovalOn, DevicesAutoUpdatesOn,
  UsersApprovalOn, NetworkFlowLoggingOn, RegionalRoutingOn,
  PostureIdentityCollectionOn (+ DevicesKeyDurationDays int). Fields that DO NOT
  exist in this version: HTTPSEnabled, ACLsExternallyManagedOn (omitted).
- RawACL has `HuJSON` (string) + `ETag` (string).
- DNSPreferences has `MagicDNS` (bool) only — NO OverrideLocal => the
  `tailscale.dns.override_local` metric is NOT emitted.
- telemetrytest.Recorder stringifies int64 log attributes as "" (Value.AsString
  on Int64 kind). int64 log attrs are still emitted correctly in production; tests
  assert their presence and verify values via the human-readable log Body. (A
  follow-up could improve the recorder to read int64 via Value.Emit/String.)
